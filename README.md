# Invoices App 💰

## 📌 About This Project

Full-Stack **Invoices App** created as learning project.  
The UI and layout are built following the official design from [Frontend Mentor](https://www.frontendmentor.io/).

**Live** version available on [Fly.io](https://invoices-app-client.fly.dev/).

---

## 🧱 Tech Stack

### 🖥️ **Client**

- **Angular 20** — Zoneless frontend framework with SSR support and Signals for fine-grained reactivity
- **TypeScript** — A typed superset of JavaScript for safer, maintainable code
- **RxJS** — Reactive programming library for handling asynchronous data streams
- **NgRx (Store + Effects)** — Predictable global state management and side-effect handling built on **RxJS**
- **Angular Forms + Zod** — Reactive form handling with schema-based, type-safe validation
- **Angular HTTPClient** — Built-in HTTP module with interceptors for secure API integration
- **Tailwind CSS** + **Sass** — Utility-first CSS with extended support for custom and complex designs
- **Motionone** — Modern animation library for the DOM with concise syntax similar to **Framer Motion** (React)
- **svg_ng_cli** — Custom **Python** CLI tool that parses SVG files into Angular components with dynamic color and size bindings

---

### 💾 **Server**

- **ASP.NET Core 9** — High-performance backend framework for building scalable REST APIs with dependency injection, middleware, and minimal APIs
- **C#** — Modern, strongly-typed programming language with async/await support, LINQ, and robust tooling
- **Entity Framework Core** — ORM for database access, schema management, migrations, and strongly-typed queries
- **PostgreSQL** — Relational SQL database used for persistent application data storage
- **Npgsql** — Official PostgreSQL provider for .NET, enabling seamless integration between ASP.NET Core and PostgreSQL
- **ASP.NET Validation** — Attribute-based request validation using Data Annotations and custom validation filters to ensure type-safe and predictable API input
- **ASP.NET Middleware** — Custom request pipeline components for logging, validation, error handling, and request processing
- **CORS (Cross-Origin Resource Sharing)** — Configured policies for secure communication between the Angular frontend and ASP.NET backend
- **Redis** — In-memory data store used for high-performance caching and API rate limiting
- **Custom Rate Limiting** — Redis-backed request throttling with configurable limits, windows, and automatic request tracking
- **Dependency Injection (DI)** — Built-in ASP.NET Core service container for managing application services and lifetimes
- **REST API Architecture** — Structured endpoints following REST principles for predictable client-server communication
- **Environment-Based Configuration** — Secure application configuration using environment variables and deployment-specific settings
- **Docker** — Containerized deployment for consistent local development and production environments
- **Fly.io** — Cloud deployment platform used for hosting and scaling the backend infrastructure
- **Cloudinary** — Cloud-based media management service used for secure image uploads, storage, optimization, and delivery
